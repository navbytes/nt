// why you'll like it

A terminal-born brain that refuses lock-in.

Most tools make you choose: a slick app that silos your data, or a pile of text files with no structure. nt gives you both — durable plain files and real structure on top.

durable by default

It's just files

todo.txt tasks + Markdown notes in one folder. Open them in any editor, grep them, git init them. No proprietary database, nothing to export, nothing to lose.

link anything

Knowledge graph

[[wikilink]] any task or note to any other. Backlinks are computed on demand by scanning files — no index to corrupt, and renames rewrite every link.

durable memory for agents

Built for AI memory

The action items and notes an agent writes today survive as plain text the next agent — or the next you — reads back tomorrow. Wire it up with a Claude Code hook, the nt mcp server, or just nt recall.

cli · tui · web

Three UIs, one store

A scriptable CLI, a live terminal UI, and an embedded web app (plus a native desktop build) — all over the exact same files, always in sync.

bring your own GUI

Obsidian-compatible

Notes are plain .md + frontmatter + [[wikilinks]]. Point an Obsidian vault at the notes/ folder and use it as your GUI while nt owns tasks and the AI loop.

pure go, no cgo

Single static binary

No CGo, no system dependencies, no runtime. curl | bash and go. Works fully offline — your notes are never on the network.

safe by construction

Git-native & lossless

nt git-init sets up merge=union so branches don't conflict on every add, and nt doctor reconciles after a merge. Every write goes through one locking, atomic, ULID-keyed engine with transactional undo/redo — a concurrent AI write is never clobbered.

// the web face — nt web

A fast SPA, compiled into the binary.

Still one static file, still fully offline. Browse the folder tree, read Markdown with wikilink navigation, and see your links as a constellation — in light or dark Tokyo Night. These screenshots follow your system theme.
